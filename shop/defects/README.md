# The four defects

These are the pre-authored bad commits Darn ships to Loose Threads during the demo. Each `<key>/patch.json` holds the full sabotaged content of exactly one shop file plus an innocent-looking commit message — the kind of small, tired-dev change that ships on a Friday. They are **staged by design but run through the real pipeline**: a defect is committed to the public repo for real, CI deploys it for real, Davis detects the regression for real, and Darn writes and opens a real fix PR. The change in each is one small hunk so the "suspect hunk" reads cleanly, and each breaks exactly its designed endpoint: `checkout-null` (AttributeError on empty-cart checkout → 500 on POST /api/checkout), `catalog-stampede` (per-item stock round-trips → latency on GET /api/catalog), `penny-shaver` (float charge drifts a cent → reconciliation fails → 500 on POST /api/pay), and `inventory-grenade` (inverted floor guard throws → 500 on POST /api/inventory/restock). Reverting a defect = restoring the original file content; nothing else changes.
